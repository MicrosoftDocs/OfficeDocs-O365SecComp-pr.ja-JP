# <a name="manage-mailbox-auditing"></a>メールボックスの監査を管理する
  
2019年1月以降、microsoft は、すべての microsoft 365 組織に対してメールボックス監査ログを既定で有効にしています。 これは、メールボックスの所有者、代理人、および管理者によって実行される特定のアクションが自動的にログに記録されることを意味します。メールボックス監査ログで検索すると、対応するメールボックス監査レコードが使用できるようになります。 既定でメールボックス監査を有効にする前に、組織内のすべてのユーザーメールボックスに対して手動で有効にする必要がありました。 

既定では、メールボックスの監査の利点は次のとおりです。

- 新しいメールボックスを作成すると、既定で監査が有効になります。 新規ユーザーに対して手動で有効にする必要はありません。 

- 監査対象のメールボックスアクションを管理することはできません。 既定では、指定された一連のメールボックスアクションがログオンの種類ごとに監査されます。 これらの[ログオンの種類](#mailbox-actions-logged-by-default)は、所有者、代理人、および管理者です。

- Microsoft によってリリースされた新しいメールボックスのアクションは、既定で監査されます。 Microsoft が新しいメールボックスアクション (特に組織を保護し、フォレンジック調査に役立つもの) をリリースすると、既定で監査されるメールボックスアクションの一覧に自動的に追加されます。 つまり、所有者、代理人、または管理者によって実行されたメールボックスアクションのリストに新しいアクションを追加する必要はありません。 

- 組織全体で一貫したメールボックスの監査ポリシーがあるように、すべてのメールボックスに対して同じアクションを監査していることを確認してください。

> [!TIP]
> 覚えておくべき重要な点は、既定ではメールボックス監査のリリースによって、メールボックスの監査を管理するために何もする必要がないということです。 ただし、詳細については、この記事の内容を参考にして、動作を既定の設定から変更するか、または完全に無効にすることをお勧めします。

## <a name="verify-mailbox-auditing-on-by-default-is-turned-on"></a>既定で [メールボックスの監査を有効にする] がオンになっていることを確認する

組織のメールボックス監査が既定でオンになっていることを確認するには、 [Exchange Online PowerShell](https://docs.microsoft.com/powershell/exchange/exchange-online/connect-to-exchange-online-powershell/connect-to-exchange-online-powershell)で次のコマンドを実行します。

```
Get-OrganizationConfig | FL AuditDisabled
``` 

**False**を指定すると、既定でオンになっているメールボックスの監査が組織で有効になります。 

既定で [メールボックスの監査] が有効になっている場合 ( *auditdisabled*プロパティが**False**に設定されている場合)、組織の設定は特定のメールボックスのメールボックスの監査設定より優先されます。 たとえば、メールボックスの*auditenabled*プロパティが**False**に設定されている場合に、既定でオンになっているメールボックスの監査が組織で有効になっている場合、そのメールボックスの既定のメールボックスアクションが監査されます。 特定のメールボックスに対してメールボックス監査が明示的に無効にされていて、それでも無効にする場合は、メールボックスの所有者とメールボックスへのアクセスが委任されている他のユーザーに対して、メールボックス監査のバイパスを設定できます。 詳細については、この記事の「[メールボックス監査ログをバイパス](#bypass-mailbox-audit-logging)する」を参照してください。

> [!NOTE]
> 既定では、メールボックスの監査が有効になっている場合、メールボックスの*auditenabled*プロパティは、現在**False**に設定されている場合は、 **True**に変更されません。 言い換えると、メールボックスの監査が既定でオンになっている場合、メールボックスの*auditenabled*プロパティは無視されます。

## <a name="mailbox-actions-logged-by-default"></a>既定でログに記録されるメールボックスの操作

メールボックスの一連の操作は、既定で次のログオンの種類ごとに記録されます。  

   - **owner** -メールボックスの所有者。 
   
   - **Delegate** -ユーザーのメールボックスに対して SendAs、sendonbehalf、または fullaccess のアクセス許可が割り当てられている別のユーザー。 ユーザーのメールボックスに対して fullaccess アクセス許可を割り当てられている管理者は、代理ユーザーとも見なされます。
   
    - **管理者**-メールボックスは、次のシナリオでのみ、管理者のログオンの種類によってアクセスされると見なされます。
    
       - 次のいずれかの Microsoft eDiscovery ツールを使用してメールボックスを検索したとき。 

         - コンプライアンスセンターでのコンテンツ検索。
       
         -  コンプライアンスセンターでの電子情報開示または高度な電子情報開示。
       
         - Exchange Online のインプレース電子情報開示。
       - [Microsoft Exchange Server MAPI エディター](https://go.microsoft.com/fwlink/p/?linkId=204086)を使用してメールボックスにアクセスするとき。

### <a name="verify-that-default-mailbox-actions-are-being-logged-for-each-logon-type"></a>ログオンの種類ごとに既定のメールボックスアクションがログに記録されていることを確認する

既定では、メールボックス監査のリリースを使用すると、 *defaultauditset*という名前の新しいメールボックスプロパティが追加されました。 このプロパティは、指定したメールボックスの各ログオンの種類について、既定のメールボックスの操作 (Microsoft が管理) を監査するかどうかを示します。 このプロパティの値を表示するには、次のコマンドを実行します。

```
Get-Mailbox <username> | FL DefaultAuditSet
```

の`Admin, Delegate, Owner`値は、3種類のすべてのログオンの種類に対する既定のメールボックスアクションが監査されており、組織の管理者が任意のログオンの種類のアクションを変更して*いない*ことを示します。 注これは、既定でオンになっているメールボックスの監査後の既定の状態が組織内で最初に有効になっていることを示しています。 

組織内の管理者が、ログオンの種類に対して監査されるメールボックスの操作 ( *auditadmin*、 *auditadmin*、または*AuditOwner*パラメーターを指定して**メールボックス**コマンドレットを使用して) を変更した場合、*defaultauditset*プロパティは、既定値とは異なります。 たとえば、の`Owner`値は、メールボックスの所有者に対する既定のメールボックスアクションのみが監査されており、の`Delegate` `Admin`アクションが変更されたことを示します。 *defaultauditset*プロパティ ( *null*値とも呼ばれます) に対して表示される値がない場合は、3つのすべてのログオンの種類のメールボックスアクションが変更されています。

監査するメールボックスの操作の変更の詳細については、この記事の「[メールボックスのアクションの変更または復元を既定でログに記録](#change-or-restore-mailbox-actions-logged-by-default)する」セクションを参照してください。

### <a name="display-a-list-of-mailbox-actions-logged-by-default"></a>既定でログに記録されたメールボックスアクションの一覧を表示する

Exchange Online PowerShell で次のコマンドを実行すると、各ログオンの種類について現在メールボックスで使用されているメールボックスの操作の一覧を表示できます。

**所有者のアクション**

```
Get-Mailbox <username> | Select-Object -ExpandProperty AuditOwner
```

**デリゲートアクション**

```
Get-Mailbox <username> | Select-Object -ExpandProperty AuditDelegate
```

**管理者アクション**

```
Get-Mailbox <username> | Select-Object -ExpandProperty AuditAdmin
```

## <a name="change-or-restore-mailbox-actions-logged-by-default"></a>既定でログに記録されるメールボックスアクションを変更または復元する

前述したように、メールボックス監査が既定で使用する主な利点の1つは、監査されるメールボックスのアクションを管理する必要がないことです。 Microsoft では、これを実行するために、新しいメールボックスアクションがリリースされたときに既定で監査されるように自動的に追加されます。 ただし、既定とは異なるメールボックスアクションのセットを監査する理由が組織にある場合があります。 このセクションでは、各ログオンの種類に対して監査されるメールボックスの操作を変更する方法と、Microsoft が管理する既定のアクションに戻す方法について説明します。

> [!IMPORTANT]
> 既定で記録されているユーザーのメールボックスの操作 (次のセクションを参照) を変更した場合、Microsoft がリリースした新しいメールボックスのアクションはそれらのメールボックスに関して監査されません。 ログオンの種類に対して監査されるアクションの一覧に、新しいメールボックスアクションを明示的に追加する必要があります。

### <a name="change-the-mailbox-actions-to-audit"></a>メールボックスのアクションを監査に変更する

ログオンの種類に応じて、監査するメールボックスの操作を変更するには、 *auditadmin*、 *auditadmin*、または*AuditOwner*パラメーターを使用して、**メールボックスの設定**コマンドレットを使用します。 これらの監査関連のパラメーターは複数値のパラメーター (つまり、複数の値を持つことができる) なので、2つの異なる方法で変更することができます。

- 既存のアクションを上書きする複数のメールボックスアクションを指定するには、 `action1,action2,...actionN`次の構文を使用します。。

- 次の構文を使用して、既存のレコードに影響を与えることなく、1つ以上`@{Add="action1","value2"}`の`@{Remove="action1","action2"}`メールボックスアクションを追加または削除することができます。または。

監査されるメールボックスの操作を変更するために使用する方法に関係なく、既定で監査されるメールボックスの操作 (変更したログオンの種類に対して) は、Microsoft によって管理されなくなります。 また、変更したログオンの種類の値は、[前に説明](#verify-that-default-mailbox-actions-are-being-logged-for-each-logon-type)した*defaultauditset* mailbox パラメーターに表示されません。

次に、これらのいずれかの方法を使用して、それぞれの種類のログオンに対してメールボックスのアクションを監査するように変更する例を示します。 

この例では、既定の操作を softdelete およびハード delete で上書きすることによって、管理者のメールボックスの操作を変更します。 

```
Set-Mailbox <username> -AuditAdmin HardDelete,SoftDelete
```

この例では、MailboxLogin owner アクションを追加します。 

```
Set-Mailbox <username> -AuditOwner @{Add="MailboxLogin"}
```

この例では、MoveToDeletedItems delegate アクションを削除します。

```
Set-Mailbox <username> -AuditDelegate @{Remove="MoveToDeletedItems"}
```

#### <a name="checking-the-defaultauditset-property"></a>defaultauditset プロパティの確認

ログオンの種類に対する既定のメールボックスアクションを変更した後、この変更を反映するために、メールボックスの*defaultauditset*プロパティが自動的に更新されます。 たとえば、メールボックス所有者`Get-Mailbox <username> | FL DefaultAuditSet`の操作を初めて追加または削除した後にを実行すると、コマンドはの`Admin, Delegate`値のみを返します。 これは、所有者に対する既定のメールボックスアクションが変更されたことを示します。これは、Microsoft がリリースした新しいメールボックス所有者のアクションがこのメールボックスに自動的に追加されないことを意味します。 管理者または代理人のログオンの種類に対するメールボックスのアクションを変更する場合も同じです。

### <a name="restore-the-default-mailbox-actions"></a>既定のメールボックスアクションを復元する

ログオンの種類に対して監査されるメールボックスの操作に変更を加えた場合は、 `Set-Mailbox -DefaultAuditSet`コマンドを実行することによって、監査される既定のメールボックスの操作を復元できます。 この操作を行うと、次の処理が行われます。

- メールボックスの操作の現在のリストは、適切なログオンの種類に対する既定のメールボックスアクションに置き換えられます。
 
- Microsoft がリリースした新しいメールボックスのアクションは、適切なログオンの種類の一覧に自動的に追加されます。

- [defaultauditset メールボックスのプロパティ](#checking-the-defaultauditset-property)は、適切なログオンの種類を含むように更新されます。

すべてのログオンの種類に対する既定のメールボックスアクションを復元するには、次のコマンドを実行します。

```
Set-Mailbox <username> -DefaultAuditSet Admin,Delegate,Owner
```

このコマンドを使用して、任意のログオンの種類に対する既定のメールボックスアクションを復元できます ( *defaultauditset*パラメーターには、**管理者**、**代理人**、または**所有者**の値を使用します)。

## <a name="turn-off-mailbox-auditing-on-by-default-for-your-organization"></a>組織に対して既定でメールボックスの監査を無効にする

何らかの理由により、組織がメールボックスの操作を監査しないことを決定した場合は、Exchange Online PowerShell で次のコマンドを実行して、組織全体のメールボックスの監査を既定で無効にすることができます。

```
Set-OrganizationConfig -AuditDisabled $true
```

既定では、組織のメールボックスの監査がオフになっている ( *auditdisabled*プロパティが**True**に設定されている) 場合、次のことが起こります。

- 組織のメールボックスの監査が無効になっています。

- メールボックスの*auditenabled*プロパティが**True**に設定されている場合でも、メールボックスの操作 (組織の監査が無効になっている時間から開始) は監査されません。

- 新しいメールボックスに対してメールボックスの監査を有効にしたり、新しい (または既存の) メールボックスで*auditenabled*プロパティを**True**に設定したりすることは無視されます。

- メールボックス監査バイパスの関連付け設定 ( **set-mailboxauditbypassassociation**コマンドレットを使用して構成) は無視されます。

- 既存のメールボックス監査レコードは、レコードの監査ログの有効期限が切れるまで保持されます。

### <a name="turn-on-mailbox-auditing-on-by-default"></a>既定でメールボックス監査をオンにする

組織のメールボックスの監査を再び有効にするには、Exchange Online の PowerShell で次のコマンドを実行します。

```
Set-OrganizationConfig -AuditDisabled $false
```

## <a name="bypass-mailbox-audit-logging"></a>メールボックス監査ログをバイパスする

現在、組織内のメールボックスの監査が既定でオンになっている場合、特定のメールボックスのメールボックス監査を無効にすることはできません。 たとえば、 *auditenabled* mailbox プロパティを**False**に設定することは無視されます。  ただし、Exchange Online PowerShell で**set-mailboxauditbypassassociation**コマンドレットを使用して、特定のユーザーによって実行されるメールボックスの操作をログに記録しないようにすることはできます。 メールボックスの監査をバイパスした場合、特定のユーザーによって実行されるメールボックスの操作は、それらのアクションが実行されているメールボックスに関係なく、監査されません。 特定のユーザーのメールボックスの監査をバイパスした場合、そのユーザーが実行したメールボックスの所有者のアクションは記録されません。 また、同じユーザーに別のユーザーのメールボックス (または共有メールボックス) へのアクセス許可が割り当てられている場合、最初のユーザーによって実行された代理人の操作もログに記録されません。

特定のユーザーのメールボックス監査ログをバイパスするには、次のコマンドを実行します。

```
Set-MailboxAuditBypassAssociation -Identity <username> -AuditByPassEnabled $true
```

指定したユーザーの監査がバイパスされていることを確認するには、次のコマンドを実行します。

```
Get-MailboxAuditBypassAssociation -Identity <username> | FL AuditByPassEnabled
```

値**True**は、そのユーザーのメールボックス監査ログがバイパスされることを示します。

## <a name="more-information"></a>詳細情報

- 既定では、メールボックス監査ログレコードは90日間保持され、削除されます。 監査ログレコードの保存期間を変更するには、「Exchange Online PowerShell の**メールボックス-auditlogagelimit** 」コマンドを使用します。 なお、メールボックス監査レコードの既定の保存期間の制限を増やすことは、Microsoft 365 監査ログのメールボックス監査ログレコードの90日の保存期間に影響を与えることに注意してください。 たとえば、メールボックス監査ログレコードの保存期間を365日に増やした場合、メールボックス監査レコードは、対応するイベントが発生した90日間のみ、Microsoft 365 監査ログで検索可能になります。 この場合は、ユーザーのメールボックス監査ログで90日より前のレコードを検索する必要があります。 メールボックス監査ログの検索の詳細については、「 [「search-mailboxauditlog](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance-audit/search-mailboxauditlog)」を参照してください。

- メールボックスの監査を有効にする前にメールボックスの*auditlogagelimit*プロパティを変更した場合、既定では、組織では、そのメールボックスの既存の監査ログの有効期限が変更されません。言い換えると、既定でオンになっているメールボックス監査は、メールボックス監査レコードの現在の保存期間制限に影響しません。

- メールボックス監査ログレコードは、各ユーザーのメールボックスの [回復可能なアイテム] フォルダーにあるサブフォルダー (*監査*という名前) に格納されます。 メールボックス監査レコードと回復可能なアイテムフォルダーについては、次の点に注意してください。
   
    - メールボックス監査レコードは、回復可能なアイテムフォルダーの記憶域のクォータに対してカウントされます。これは既定で30GB ます (警告クォータは 20 GB です)。 メールボックスに保留が設定されている場合、またはメールボックスがコンプライアンスセンターのアイテム保持ポリシーに割り当てられている場合は、回復可能なアイテムフォルダーの記憶域のクォータが自動的に 100 GB (90 MB の警告クォータ) に増加します。

    - メールボックス監査レコード[は、回復可能なアイテムフォルダーのフォルダー制限](https://docs.microsoft.com/office365/servicedescriptions/exchange-online-service-description/exchange-online-limits#mailbox-folder-limits)にもカウントされます。 audit サブフォルダーに格納できるアイテムの最大数 (この場合は、この場合は監査レコード) は300万アイテムです。 

      > [!NOTE]
      > 既定では、メールボックスの監査が、回復可能なアイテムフォルダーの記憶域のクォータまたはフォルダーの制限に影響を与える可能性はほとんどありません。 

    - Exchange Online の PowerShell で次のコマンドを実行すると、回復可能なアイテムフォルダーの監査サブフォルダー内のアイテムのサイズと数を表示できます。 
       ```
       Get-MailboxFolderStatistics <username> -FolderScope RecoverableItems | Where-Object {$_.Name -eq 'Audits'} | FL FolderPath,FolderSize,ItemsInFolder
       ```

     - [回復可能なアイテム] フォルダー内の監査ログレコードに直接アクセスすることはできません。代わりに、 **「search-mailboxauditlog**コマンドレットを使用するか、Microsoft 365 監査ログを検索して、メールボックス監査レコードを検索して表示します。

- メールボックスが保留になっている場合、またはコンプライアンスセンターでアイテム保持ポリシーに割り当てられている場合、メールボックスの*auditlogagelimit*プロパティによって定義されている期間 (既定では90日間に設定されます) の監査ログレコードが引き続き保持されます。 保留中のメールボックスの監査ログレコードを長期間保持するには、 *auditlogagelimit*プロパティの値を大きくして、保持期間を延長する必要があります。
