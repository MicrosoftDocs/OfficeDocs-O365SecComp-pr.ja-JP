# <a name="supervision-policies-in-office-365"></a>Office 365 の監督のポリシー

Office 365 の監督ポリシーを使用すると、指定した校閲者による、従業員の通信をキャプチャできます。内部と外部の電子メール、マイクロソフトのチーム、または組織内のサード パーティ製の通信をキャプチャする特定のポリシーを定義することができます。校閲者、組織のメッセージの標準に準拠しているかどうかを確認するメッセージを確認し、分類型を解決します。これらのポリシーは、メッセージ データ、および規制の強制 & 罰金のリスク量の増加、通信チャネルの種類の増加を監視するなど、多くの近代のコンプライアンスに関する課題を克服するにも役立ちます。

一部の組織では、IT サポート、コンプライアンスの管理グループ間での業務の分離が可能性があります。Office 365 は、監督のポリシー機能をサポートし、ポリシーの構成を持つテナントを構成して、キャプチャされた通信に作用している間の分離をサポートします。たとえば、組織の IT グループが構成され、組織のコンプライアンス ・ チームによって管理されている監視ポリシーをサポートするために、ロールのアクセス許可とグループの設定を担当する可能性があります。

## <a name="scenarios-for-supervision-policies"></a>監督ポリシーのシナリオ

監督のポリシーは、組織は、いくつかの領域での通信を監視に役立ちます。

- **企業ポリシー**

    従業員は、利用規約、倫理的な標準、およびすべてのビジネスに関連する通信の他の企業のポリシーに従う必要があります。監督のポリシーは、ポリシー違反を検出し、これらの種類の問題を軽減するように修正処理を実行できます。などの迷惑行為や、従業員の通信では、不適切、または不快感を与える言語の使用などの人事管理違反の可能性を組織を監視する可能性があります。

- **リスク管理**

    組織は、すべての通信のインフラストラクチャと企業のネットワーク ・ システム全体に分散する必要があります。監督ポリシーを使用して特定し、潜在的な法的リスクおよびリスク管理は、企業のオペレーションを破損することができます前に、リスクを最小限に抑えるに役立ちます。たとえば、今後の買収、合併、利益の開示を行い、再編成、リーダーシップ チームの変更などの機密性の高いプロジェクトの不正な通信を組織を監視する可能性があります。

- **規制へのコンプライアンス**

    ほとんどの組織は、標準運用手順の一環として、法令遵守の種類でに従う必要があります。これらの規制には多くの場合監督のいくつかの型を実装する組織が必要とする、またはメッセージングの監視プロセスは、業界に適切です。3110 金融業界の規制機関 (FINRA) ルールでは、従業員の活動を監視する監視の手順が組織の要件の例の種類のビジネスを行っています。別の例には、潜在的なマネー ・ ロンダリング、内部者取引、複数のパスワード ポリシー、または買収活動を防ぐため、組織のブローカー ディーラーを監視する必要がある可能性があります。監督のポリシーは、組織のプロセスをモニターと企業のコミュニケーション上のレポートの両方に提供することでこれらの要件を満たすに役立ちます。

監督ポリシーを使用する通信プラットフォームを次のいずれかのメッセージを監視できます。

- **電子メールの交換:** Office 365 サブスクリプションの一部として、Exchange のオンラインでホストされているメールボックスは、すべて、メッセージの監視の対象となります。電子メールおよび添付ファイルの監視ポリシーの条件に一致するが即座に監視と監督レポートです。監督のサポートされている添付ファイルの種類は、 [Exchange メール フロー ・ ルール ・ コンテンツの検査用にサポートされているファイルの種類](https://docs.microsoft.com/exchange/security-and-compliance/mail-flow-rules/inspect-message-attachments#supported-file-types-for-mail-flow-rule-content-inspection)と同じです。
- **マイクロソフト チーム:** チャットの通信、パブリックとプライベートのマイクロソフト チーム チャンネルと個々 のチャットの両方に関連付けられている添付ファイルを管理できます。監督のポリシーの条件に一致するチーム チャットでは、24 時間に 1 回の処理や、[監視および監督のレポートでは。
- **サード パーティのソース:** 組織で Office 365 のメールボックスにこのデータをインポートした場合は、サード パーティのソース (Facebook のドロップ ボックスのような) からの通信を監督できます。[Office 365 にサード パーティ製のデータをインポートする方法について説明](https://docs.microsoft.com/office365/securitycompliance/archiving-third-party-data)します。

- 準拠や非準拠の問題があると 1 つのアイテムにタグ付けします。タグ付けの動作を明確化する項目にコメントを記録することもできます。
- 一括は、準拠や非準拠の問題があると複数の項目をタグ付けします。タグ付けの動作を明確にするために複数の項目にコメントを記録することもできます。
- アイテム、日付と時間のアクション、解像度のタグ、および含まれているコメントの解決者を含む、1 つのアイテムにタグ付けの履歴を表示します。
- 準拠や非準拠の問題があると既に確認済みの項目を再分類します。1 つまたは複数の項目を再分類の動作を明確にするためにコメントを記録することもできます。

#### <a name="resolved-items-tab"></a>[アイテム] タブの解決

**項目の解決**] タブでは、校閲者が選択したポリシーに既に解決されたすべてのアイテムを表示できます。ここでは次のようなことができます。

- 迅速に表示し、件名、送信者、および解決済みのアイテムの日付を並べ替えます。
- 選択したアイテムのクラス分けとコメントの履歴を表示します。

## <a name="reporting"></a>レポート

監督レポートを使用して、ポリシーとレビュー担当者のレベルでのレビュー」アクティビティを参照してください。ポリシーごとに、レビュー」アクティビティの現在の状態で、ライブの統計情報を表示することも。監督レポートを使用できます。
  
- ポリシーが意図したとおりに動作していることを確認します。
- 確認のため、多くの通信を確認が識別されます。
- 通信が準拠していないし、どれが成功していることが数を確認することを確認します。この情報は、ポリシーを微調整または校閲者の数を変更するかどうかを決定するのに役立ちます。

### <a name="view-the-supervision-report"></a>監督レポートを表示します。

1. 監督レポートを表示するアクセス許可を持つ、Office 365 の組織の管理者アカウントの資格情報を使用して[セキュリティ & コンプライアンス センター](https://protection.office.com/)にサインインします。
2. いずれかの**レポート**を参照して\>**ダッシュ ボード**または**監視**します。ウィジェットが現在の監督のポリシー利用状況の概要を報告する監督が表示されます。
3. **監督**ウィジェットを開くには、詳細なレポートのページを選択します。

> [!NOTE]
> **レポート**] ページにアクセスできない場合は、監督機関による監査の役割グループのメンバーであることの手順に従って確認[監督は、組織で使用可能な](configure-supervision-policies.md)のです。このロール グループで作成し、監督を管理するに含まれているポリシーし、レポートを実行します。
  
### <a name="how-to-use-the-report"></a>レポートを使用する方法

Outlook と Outlook での校閲者の監視フォルダーに電子メールが配信される監視ポリシーでは、確認のための通信メッセージを識別するときに web アプリケーションです。このレポートには、確認プロセスの各段階での各ポリシーの名前との通信の数が一覧表示されます。
  
レポートを使用します。
  
- すべてのデータを表示、または特定のポリシーです。
- 準拠、Questionable など) などのタグの種類、レビュー担当者、またはメッセージの種類によってグループ化されたデータを表示します。
- アクティビティの日付、ポリシー、およびレビュー担当者の活動によってに基づいて CSV ファイルにデータをエクスポートします。
- アクティビティの日付、タグの種類、レビュー担当者、およびメッセージの種類に基づいてデータをフィルター処理します。

**タグの種類**] 列に表示される値の内訳を次に示します。
  
|**タグの種類**|**意味**|
|:-----|:-----|
| 確認していません | まだ確認されていない電子メールの数。これらの電子メールは、確認または Outlook または Outlook Web App での校閲者の監視フォルダーに Office 365 の監視ダッシュ ボードを待機しています。|
| 準拠 | 電子メールの数は、確認し、準拠としてマークします。これらのメッセージでは、解決する必要があります。 |
| 問題があります。 | 電子メールの数は、確認し、疑わしいとマークします。フラグとして機能し他のレビュー担当者に役立つことができます電子メールがコンプライアンスのための調査を必要があるかどうかを確認します。これらのメッセージでは、解決する必要があります。 |
| 非準拠 (アクティブ) | 校閲者について現在調査中は、非準拠の電子メールの数。 |
| 非準拠 (解決) | 校閲者を調査し、解決する非準拠の電子メールの数。 |
| ポリシーをクリックします。 | 合計数 (日単位) の交換、チーム、および監視ポリシーで定義されている 1 つまたは複数の条件に一致したサード ・ パーティ製のデータ ソースからのメッセージ |
| 範囲内 | 合計数 (日単位) の交換、チーム、およびサード パーティのデータ ソースの監視ポリシーによってスキャンされたメッセージ |
| 解決 | 交換、チーム、および**解決済み**として分類されているサードパーティのデータ ソースからのメッセージの合計数|

> [!NOTE]
> このレポートに表示される前に、監督のポリシーを準備する必要があります最初。さらに、ポリシーが削除されると、履歴データが表示されます。ただし、「が存在しないポリシー」として示されていることと、**エクスポート**関数は使用できません。

## <a name="auditing"></a>監査

規制情報を提供する必要がありますいくつかの場合、または従業員の活動との通信の監視を証明するためのコンプライアンス監査役です。定義済みのポリシーまたは監督のポリシーが変更または更新されるたびに関連付けられたすべての監督活動の概要があります。監督のポリシーには、内部または外部の監査の完了の準備のための組み込みの監査記録があります。監督手順の証拠には、監督のポリシーで監視するすべてのアクションの詳細な監査履歴を示しています。

次の監督ポリシー アクティビティでは、監査し、統合された Office 365 の監査ログを使用して表示することができます。

|**アクティビティ**|**関連コマンド**|
|:-----|:-----|
| ポリシーを作成します。 | 新しい-SupervisoryReviewPolicy <br> 新しい-SupervisoryReviewRule |
| ポリシーを編集します。 | セット SupervisoryReviewPolicy <br> セット SupervisoryReviewRule |
| ポリシーを削除します。| 削除 SupervisoryReviewPolicy |

統一された監査ログの検索機能を使用して監査を取得することができますか[検索 UnifiedAuditLog](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance-audit/search-unifiedauditlog)の PowerShell コマンドレットを使用しています。

たとえば、次の例では、すべて、監督機関による監査アクティビティ (ポリシーおよび規則) の活動を返し、ごとに詳細な情報を一覧表示。

```
Search-UnifiedAuditLog -StartDate $startDate -EndDate $endDate -RecordType DataGovernance -ResultSize 5000 | Where-Object {$_.Operations -like "*SupervisoryReview*"} | fl CreationDate,Operations,UserIds,AuditData 
```

監督レポートとログに記載されている情報に加えポリシー活動のすべての監視の詳細な一覧を取得する[Get SupervisoryReviewActivity](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance/get-supervisoryreviewactivity?view=exchange-ps) PowerShell コマンドレットを使用することもできます。
