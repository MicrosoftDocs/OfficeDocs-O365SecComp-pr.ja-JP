## <a name="how-office-365-message-encryption-works"></a>Office 365 のメッセージの暗号化のしくみ

Office 365 のメッセージの暗号化は、Azure の情報保護の一部である Microsoft Azure 著作権管理 (Azure RMS) に組み込まれているオンライン サービスです。Office 365 の管理者は、暗号化するための条件を決定するのには、メール フロー ルールを定義できます。たとえば、ルールでは、特定の受信者宛てのすべてのメッセージの暗号化を要求できます。
  
Exchange Online の暗号化メールの流れの規則に一致する電子メール メッセージを送信ユーザーと送信される前にメッセージが暗号化されます。メールを Outlook クライアントを使用するすべての Office 365 ユーザーには、ネイティブな送信者と同じ組織内に含まれていない場合でも、暗号化および権利が保護されたメールの経験を読む最初のクラスが表示されます。Outlook デスクトップで、Outlook の Mac、iOS および Android、モバイルの Outlook と Outlook Web App がサポートされている Outlook クライアントに含まれます。
  
Outlook.com、Gmail、yahoo アカウントに送信された暗号化、または権利が保護されたのメールを受信する暗号化されたメッセージの受信者は、自分の Microsoft アカウントまたは Gmail や yahoo の資格情報を使用してホーム ポータルに簡単に認証できます。
  
権利で保護された、または暗号化されたメールを Outlook 以外のクライアントであるエンド ・ ユーザーは、受信した暗号化および権利が保護されたメッセージを表示するのにはホームのポータルを使用します。
  
メッセージおよび添付ファイルを暗号化するには Office 365 のメッセージの暗号化を使用してのサイズの制限が大きくなることです。制限の詳細についてを参照してください[Exchange オンライン制限します](https://technet.microsoft.com/en-us/library/exchange-online-limits.aspx)。
  
## <a name="defining-rules-for-office-365-message-encryption"></a>Office 365 Message Encryption のルールの定義
<a name="Rules"> </a>

メール フロー ルールを定義するのには、オンラインの Exchange および Exchange のオンライン保護の管理者が、Office 365 のメッセージの暗号化の新しい機能を有効にする方法の 1 つです。これらのルールは、どのような条件の e メールでメッセージを暗号化するかを決定します。暗号化アクションをルールに設定すると、ルールの条件に一致するすべてのメッセージは、送信することをする前に暗号化されます。
  
メール フロー ルールは、柔軟性があり、1 つの規則で特定のセキュリティ要件を満たすことができますので、条件を結合すること。たとえば、指定されたキーワードが含まれているし、外部の受信者にアドレス指定されるすべてのメッセージを暗号化するためにルールを作成できます。Office 365 のメッセージの暗号化のための新しい機能では、暗号化された電子メールの受信者からの返信も暗号化します。
  
メール ホームの新機能を利用するのには、フロー ルールを作成する方法の詳細については、 [Office 365 のメッセージの暗号化のための規則の定義](define-mail-flow-rules-to-encrypt-email.md)を参照してください。
