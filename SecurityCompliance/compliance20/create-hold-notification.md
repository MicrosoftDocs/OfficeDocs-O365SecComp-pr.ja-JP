# <a name="create-a-legal-hold-notice"></a>法的保持義務通告を作成します。

保管担当者通信の高度な電子的証拠開示 (プレビュー) を使用すると、組織は通告との通信中のワークフローを管理できます。通信ツールでは、法務部門体系的に送信、収集、および追跡できます法的保持義務通告。柔軟な作成処理を保留中の通知ワークフローおよび通告に送信される通知の内容をカスタマイズすることもできます。 

資料では、保留中の通知ワークフローの手順を説明します。

## <a name="step-1-specify-communication-details"></a>手順 1: 通信の詳細を指定します。

最初のステップでは、法的保持義務通告の適切な詳細またはその他の管理者の通信を指定します。 

1. セキュリティ & コンプライアンス センターでは、組織のサポート案件の一覧を表示するのには、**電子的証拠開示 _gt 高度な電子情報開示 (プレビュー)** に移動します。
   
2. [**通信**] タブをクリックし、**新しい通信**] をクリックします。
   
3. **名通信**] ページで、次の (必須) の通信の詳細を指定します。

    - **名前**: これは、通信の名前です。
    
    - **発行責任者**: ドロップダウン ・ リストには、大文字のメンバーの一覧が表示されます。通告に送信される各通知は、指定した発行責任者の代理として送信されます。

4. [ **次へ**] をクリックします。

## <a name="step-2-define-the-portal-content"></a>ポータルのコンテンツを定義する手順 2。

次に、作成し、保留中の通知のコンテンツを追加できます。**通信の作成**ウィザードで [**ポータル コンテンツの定義**] ページで、[保留中の通知の内容を指定します。このコンテンツの発行、再発行、アラーム、およびエスカレーションの通知を自動的に追加されます。さらに、このコンテンツは、管理者のコンプライアンスのポータルに表示されます。 

ポータルのコンテンツを作成します。

1. 型 (またはカットと別のドキュメントから paster) ポータルのコンテンツのテキスト ボックスに、保留リストを確認します。 

2. 通知をカスタマイズし、保管担当者コンプライアンス ポータルを共有する、通知データの差し込みの変数に挿入します。

3. [ **次へ**] をクリックします。

  >[!Tip]
  >コンテンツおよびポータルのコンテンツの形式をカスタマイズする方法の詳細については、[通信のエディターを使用して](using-communications-editor.md)参照してください。

## <a name="step-3-set-the-required-notifications"></a>ステップ 3: 必要な通知を設定します。

保留中の通知の内容を定義した後、送信すると、通知プロセスを管理するワークフローを設定できます。通知は、通知するために送信される電子メール メッセージおよび通告でのフォロー アップです。通信に追加されたすべての管理者は、同じ通知で表示されます。 

設定し、保留中の通知を送信、発行、再発行、組み込み、通知をリリースします。

### <a name="issuance-notification"></a>発行通知 

通信を作成すると後、は、**発行通知**が、指定した発行責任者によって開始されます。発行通知は、最初の通信、情報保持義務について知らせるために、管理者に送信します。 

発行、通知を作成します。

1. **発行**] タイルで [**編集**] をクリックします。
   
2. 必要に応じて、 **[cc]** および **[bcc]** フィールドに、ケースの他のメンバーやスタッフを追加します。これらのフィールドに複数のユーザーを追加するに電子メール アドレスをセミコロンで区切ります。
   
3. (必要な) 通知の**件名**を指定します。
   
4. 内容 (必須) の管理者に提供するには追加の説明を指定します。発行通知のためには、手順 2 で定義したポータルのコンテンツが追加されたことに注意してください。 
   
5. **[保存]** をクリックします。 

### <a name="re-issuance-notification"></a>再発行通知 

ケースの進行に合わせて通告が指示が以前よりも、追加、または以下のデータを保持するために必要な可能性があります。保留中の通知の内容を更新した後、再発行通知は、情報保持義務への変更に関する通告を警告します。

再発行通知を作成します。 

1. **再発行**] タイルで [**編集**] をクリックします。
   
2. 必要に応じて、 **[cc]** および **[bcc]** フィールドに、ケースの他のメンバーやスタッフを追加します。これらのフィールドに複数のユーザーを追加するに電子メール アドレスをセミコロンで区切ります。
   
3. (必要な) 通知の**件名**を指定します。
   
4. 内容 (必須) の管理者に提供するには追加の説明を指定します。再発行通知のためには、手順 2 で定義したポータルのコンテンツが追加されたことに注意してください。
   
5. **[保存]** をクリックします。

>[!Note]
>保留中の通知が変更された場合、通知に割り当てられているすべての通告を再発行の通知を自動的に送信されます。通知が送信されると、通告は、保留中の通知を再度確認するのには求められます。アラームまたはエスカレーション ワークフローを設定すると、これらも再開始されます。 

### <a name="release-notification"></a>リリースの通知

問題が解決された後、または、管理者は、不要になった場合は、コンテンツを保持する、ケースから管理者を解放することができます。書以前に発行された保留中の通知をする場合は、それぞれの義務がリリースされた通告を警告するリリースの通知を使用できます。

リリースの通知を作成します。 

1. **リリース**] タイルで [**編集**] をクリックします。
   
2. 必要に応じて、 **[cc]** および **[bcc]** フィールドに、ケースの他のメンバーやスタッフを追加します。これらのフィールドに複数のユーザーを追加するに電子メール アドレスをセミコロンで区切ります。
   
3. (必要な) 通知の**件名**を指定します。
   
4. 内容 (必須) の管理者に提供するには追加の説明を指定します。
   
5. [**保存**] をクリックし、次の手順に進みます。 

## <a name="optional-step-4-set-the-optional-notifications"></a>(省略可能)手順 4: 省略可能な通知を設定します。

必要に応じて、次応答を停止の通告を作成およびスケジュールの自動通知、エスカレーション、通知のワークフローを簡素化できます。

### <a name="reminders"></a>事前通知

保留中の通知を送信した後、応答を停止の通告を通知ワークフローを定義することでフォロー アップをできます。 

アラームをスケジュールします。

1. **アラーム**] タイルで [**編集**] をクリックします。
   
2. (必要な)**状態**の切り替えを有効にして、**通知**のワークフローを有効にします。
   
3. (必須) の**アラーム間隔 (日数)** を指定します。これは、フォロー アップし、最初のアラームの通知を送信するまでの日数です。などの 7 日にアラームの間隔を設定する場合、最初のバルーンは送信保留通知が最初に発行された後、7 日間。以降のすべてのアラームは、7 日にも送られます。
   
4. (必要な)**通知の数**を指定します。このフィールドは、通告をされていない応答を送信する数のアラームを指定します。などの事前通知の数を 3 に設定すると、管理者は最大 3 つのアラームが表示されます。保管担当者は、保留中の通知を確認後、そのユーザーに通知が送信されなく。
   
5. (必要な) 通知の**件名**を指定します。 
   
6. 内容 (必須) の管理者に提供するには追加の説明を指定します。アラーム通知の最後に手順 2 で定義したポータルのコンテンツが追加されたことに注意してください。
   
7. [**保存**] をクリックし、次のステップです。

### <a name="escalations"></a>エスカレーション 

状況によっては、応答を停止の通告を別の方法でフォロー アップをする必要があります。保管担当者は、アラームの指定した数を受信した後の保留中の通知を確認しない、法務チームは、管理者およびマネージャーに、エスカレーションの通知を自動的に送信するワークフローを指定できます。

エスカレーションをスケジュールします。

1. **エスカレーション**] タイルで [**編集**] をクリックします。
   
2. **ステータス**の表示/非表示を有効にして、**エスカレーション**のワークフローを有効にします。
   
3. (必須)、**エスカレーションする間隔 (日数)** を指定します。 
   
4. **エスカレーションの数**(必須) を指定します。このフィールドは、通告をされていない応答を送信する方法の多くのエスカレーションを指定します。などのエスカレーションの回数を 3 に設定するとし、エスカレーションの通知は送信書や上長に最大 3 回。保留中の通知に同意するもの、保管担当者後のエスカレーションは送信されません。 
   
5. (必要な) 通知の**件名**を指定します。 
   
6. 内容 (必須) の管理者に提供するには追加の説明を指定します。エスカレーションの通知のためには、手順 2 で定義したポータルのコンテンツが追加されたことに注意してください。
   
7. [**保存**] をクリックし、次のステップです。
   
## <a name="step-5-assign-custodians"></a>通告を割り当てる手順 5。 

通知の内容を完成した後に通知を送信する通告を選択します。 

通告を追加します。

1. 名の横のチェック ボックスをクリックして、通信に通告を割り当てます。

    通信が作成されると、通知ワークフローを選択した通告を自動的に適用されます。
   
2. **次**の通信設定と詳細を確認する] をクリックします。
 
>[!NOTE]
>通告の場合に追加されているし、するいないと、ケース内の別の通知が送信されたユーザーのみを追加できます。

## <a name="step-6-review-settings"></a>手順 6: 設定の確認

設定を確認し、**送信**の通信を完了するのにをクリックすると、システムは自動的に開始されている発行通知を送信することによって通信のワークフローを使用します。
