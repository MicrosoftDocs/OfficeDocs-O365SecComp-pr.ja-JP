# <a name="manage-gdpr-data-subject-requests-with-the-dsr-case-tool-in-the-office-365-security-amp-compliance-center"></a>DSR の case ツールを Office 365 のセキュリティでの GDPR データ主題の要求を管理する&amp;コンプライアンス センター

EU 全般的なデータ保護規制 (GDPR) は、保護し、個人のプライバシーの権利、欧州連合 (EU) の内部を有効にすることです。欧州連合の GDPR は個人 (と呼ばれるデータ項目) にアクセス、取得、修正、消去、および、個人データの処理を制限する権利。GDPR の下では、個人データは、特定された、または識別可能な自然人に関連する情報を意味します。データのサブジェクトの要求または DSR は、自分の組織のユーザーによって、個人データにアクションを実行するのには正式な要求が呼び出されます。DSRs への応答は、Office 365 のデータの詳細については、 [Office 365 のデータ主題を要求ガイド](https://go.microsoft.com/fwlink/?linkid=871169 )を参照してください。

## <a name="step-3-run-the-search-query"></a>手順 3: 検索クエリを実行します。

DSR のケースを作成し、メンバーを追加した後は、サポート案件に関連付けられている組み込みの検索を実行します。この既定の検索クエリは、次を行います。
  
- データのサブジェクトによって送受信されたすべてのメール アイテムを組織内のすべてのメールボックスを検索します。これは電子メール メッセージ内のすべてのユーザー フィールドにデータの件名を検索する*参加者*の電子メール プロパティを使用します。このプロパティは、データのサブジェクト******に** **[cc]**、および **[bcc]** フィールドではアイテムを返します。オンライン Exchange 内のパブリック フォルダーは、メッセージ データのサブジェクトによって送信または受信をも検索されます。 
    
- ドキュメントおよびアイテムの作成またはデータのサブジェクトによってアップロードされたことを組織内のすべてのサイトを検索します。これは、次のサイト プロパティを使用して行われます。
    
  - *Author*プロパティは、Office ドキュメントで、[作成者] フィールドのデータの件名が一覧表示される項目を返します。ドキュメントがコピーされ、他のユーザーによってアップロードされた場合でも、この値は永続化します。 
    
  - *CreatedBy*プロパティは、作成またはデータのサブジェクトによってアップロードされた項目を返します。 
    
ここでは、DSR のケースを作成するときに自動的に作成される組み込みの検索のキーワード クエリがどのようにします。
  
```
participants:"<email address>" OR author:"<display name>" OR createdby:"<display name>"
```

たとえば、データのサブジェクトの名前が Ina Leonte の場合は、キーワード クエリになります次のように。
  
```
participants:"ina@contoso.com" OR author:"Ina Leonte" OR createdby:"Ina Leonte"
```

## <a name="optional-step-5-revise-the-built-in-search-query"></a>(省略可能)手順 5: 組み込みの検索クエリを変更します。

組み込みの検索を実行した後、以下の検索結果を返すには、スコープを絞り込むには、変更できます。クエリに条件を追加することによってこれを行うことができます。条件は**AND**演算子によってキーワード クエリに論理的に接続します。項目は、キーワード クエリと追加する任意の条件の両方を満たす必要があります、検索結果に返されるを意味します。これは、結果を絞り込む条件がどのように支援します。(別のプロパティを指定する条件) の検索クエリに 2 つ以上の一意の条件を追加する場合は、 **AND**演算子でこれらの条件が論理的に接続します。つまり (キーワード クエリ) の他のすべての条件を満たすアイテムだけが返されます。1 つの条件に複数の値を (カンマまたはセミコロンで区切る) を追加すると、これらの値は**OR**演算子で接続されています。つまり、含まれている場合、条件のプロパティに指定された値のいずれかの項目が返されます。 
  
DSR ケースの組み込みの検索クエリに追加することの条件のいくつかの例を次に示します。検索クエリで使用される実際のプロパティの名前は、かっこで表示されます。
  
- **ファイルの種類 ( `filetype`)** -ドキュメントまたはファイルの拡張子を指定します。ドキュメントや Word、Excel、および OneNote など、特定の Office アプリケーションで作成されたファイルを検索するには、この条件を使用します。 
    
- **メッセージの種類 ( `kind`)** -を検索する電子メール アイテムの種類を指定します。構文を使用することができますたとえば、`kind:email OR kind:im`ビジネス会話またはマイクロソフトのチームでの 1 対 1 のチャットの電子メール メッセージ、Skype のみを返します。 
    
- **コンプライアンスのタグ (`compliancetag`)** -電子メール メッセージやドキュメントに割り当てられているラベルを指定します。この条件では、特定のラベルには、分類の項目を返します。電子メールとデータ管理のためのドキュメントを分類し、ラベルで定義されている分類に基づく保存ルールを適用するラベルを使用します。これは、組織を使用して、ラベルのデータのプライバシーに関連するコンテンツを分類するか、または個人データや機密情報を含むために、DSR の調査に有用な条件です。この条件の値をワイルドカードで完全なラベル名またはラベル名の最初の部分を使用します。詳細については、 [Office 365 でのラベルの概要](labels.md)を参照してください。
    
一覧と DSR の case ツールで使用可能なすべての条件の説明は、「キーワード クエリとコンテンツ検索の検索条件」資料の[検索条件を](keyword-queries-and-search-conditions.md#search-conditions)参照してください。

### <a name="exporting-partially-indexed-items"></a>部分的にインデックス付きのアイテムをエクスポートします。

DSR の新しいケースを作成するときに作成される組み込みの検索から部分的にインデックス付きの項目 (インデックスの項目とも呼ばれます) をエクスポートしないことをお勧めします。ですので、検索、結果以上の場合は、部分的に他のユーザーを組織内の項目のインデックスを作成し、データのインデックス項目のない部分だけ)。代わりに、DSR の場合はデータの対象に関連する部分的にインデックス付きのアイテムのみをエクスポートするように設計されているに関連付けられている独立したコンテンツの検索を作成することをお勧めします。 
  
ここでは、部分的にインデックス付きのアイテムをエクスポートするのには高度なプロセスです。エクスポートして後、に、アイテムは DSR アクセスへの応答があるかどうかを確認したり、要求をエクスポートできます。
  
1. DSR のケースを開き、[**検索**] ページで、新しい検索を作成します。 
    
2. 検索する検索クエリとコンテンツの場所を構成するための次の条件を使用します。
    
    - 空または空白のキーワード クエリを使用します。検索対象となるコンテンツの場所のすべてのアイテムが返されます。
    
    - データの対象者の Exchange Online のメールボックスのみが OneDrive のアカウントを検索します。
    
3. 検索を実行すると、それが完了すると、エクスポートし、([手順 4](#step-4-export-the-data)で説明した) ように、検索結果をダウンロードできます。部分的にインデックス付きのアイテムをエクスポートするのにには、次の設定を使用します。 
    
    - [**出力オプション**] では、部分的にインデックス付けされたアイテムのみをエクスポートするのには 3 番目のオプション (**認識できない形式では、ある項目だけが暗号化されているか他の理由によりインデックスが作成されませんでした**) を選択します。
    
    - [**エクスポートする Exchange のコンテンツとして**、任意のオプションを任意に設定を選択します。 
    
    - **SharePoint ドキュメントのバージョンを含める**] オプションを選択する場合は、バージョンのインデックスは部分的にドキュメントのバージョンがエクスポートされます。 
    
部分的にインデックス付きのアイテムの詳細についてを参照してください。

### <a name="searching-and-exporting-data-from-microsoft-teams-and-office-365-groups"></a>検索し、マイクロソフトのチームと Office 365 のグループからデータをエクスポートします。

マイクロソフトのチーム (チーム チャットや 1 対 1 のチャットと呼ばれる) の [チャット] ボックスの一覧に含まれている会話は、Exchange のオンライン、チャットに参加するユーザーのメールボックスに格納されます。また、1 対 1 チャットのユーザーが共有ファイルは、ファイルを共有するユーザーの OneDrive アカウントに格納されます。組み込みの検索は、組織のすべてのメールボックスおよび OneDrive のアカウントを検索するためチームのチャットとチャットのセッション (データのサブジェクトは、作成またはアップロード) で共有されるドキュメントは、DSR の場合は、組み込みの検索によって返されます。
  
代わりに、チームのチャネル (チャネルのメッセージとも呼ばれます) の一部である会話は、チームに関連付けられているメールボックスに格納されます。マイクロソフトのチームに関連付けられているすべてのメールボックスを検索するためにも、これらの会話に含まれるデータのサブジェクトの種類が組み込みの検索で返されます。さらに、チームのチャネルの共有可能性がありますデータのサブジェクトのタイルは、チームの SharePoint サイトに格納されます。ファイルが作成または uploadedby データのサブジェクトによって返される、DSR の場合は、組み込みの検索マイクロソフト チームに関連付けられているサイトが検索対象に含まれているためです。
  
同様に、メールボックスと、Office 365 のグループに対応する SharePoint サイトにも付属して組み込みの検索します。つまり、データのサブジェクトによって送信または受信する電子メール メッセージを作成またはデータのサブジェクトによってアップロードされたファイルが返されます。 
  
詳細については、マイクロソフトのチームと Office 365 のグループ内のアイテムを検索するか、メンバーの一覧を取得する方法については、コンテンツの検索を使用して、 [Office 365 でのコンテンツの検索](content-search.md#searching-microsoft-teams-and-office-365-groups)に検索マイクロソフト チームと Office 365 の「グループ」を参照してください。
