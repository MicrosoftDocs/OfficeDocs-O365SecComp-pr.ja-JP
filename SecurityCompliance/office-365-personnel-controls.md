# <a name="office-365-personnel-controls"></a>Office 365 での担当者の制御 

人事審査は、ユーザーの過去の動作と状態を確認して検証するプロセスであり、Office 365 サービスが侵害されないようにするための重要な対策管理です。 これまでの動作は、ユーザーの今後の動作を正確に予測するものではありませんが、潜在的な不正な俳優を特定するのに役立ちます。 microsoft の人事審査標準は、政府機関またはコマーシャルクラウドのお客様へのオンラインサービスの開発、運用、または提供に携わるすべての microsoft 従業員、インターン、および臨時スタッフに適用されます。 Microsoft によって運用されていない国内のクラウド環境の審査標準は、特定の環境ごとに、運用パートナーの担当者によって定義されます。

## <a name="microsofts-personnel-screening-standard"></a>Microsoft の人事審査標準

microsoft の従業員の審査プラクティスは、microsoft の企業標準および米国国立標準テクノロジ (NIST) によって管理されています。 以下にアクセスする必要がある microsoft スタッフは、microsoft の従業員審査標準に従います。
- データセンター、コロケーション、セキュリティで保護されたルーム、cages、サーバーラック、または商用クラウドのお客様向けのオンラインサービスをサポートするインフラストラクチャを提供するエッジサイトへの物理的なアクセス。
- 特定の管理環境を通じて提供される政府機関またはコマーシャルクラウドの顧客データへの論理的なアクセス。
    - 政府機関またはコマーシャルクラウドの顧客データを転送または保存するデバイスおよびサービスへのネットワーク管理アクセス。

スクリーン処理要件をトリガーする特定の個人関連イベントは次のとおりです。
- 新しい Microsoft スタッフは、審査が定義された要件である役割に配置されました。
- 内部の Microsoft スタッフは、定義済みの要件として、現在審査を含んでいる既存の役割に転送または移動します。
- 定義済みの要件として審査を含めるようにスコープを変更する既存の役割。

## <a name="screening-enforcement-criteria"></a>審査の実施条件

許可された担当者のみが、審査を必要とする顧客データまたは環境にアクセスできるようにするには、次の適用条件を適用します。

**米国のクラウド環境のみ**:
- 審査を必要とする顧客データまたは環境へのアクセスは、adjudication が完了し、スクリーン処理の要件が満たされた後にのみ許可される必要があります。
- お客様のデータまたは関連する環境へのアクセスを必要としなくなった microsoft のスタッフは、microsoft を離れるか新しい役割に移行する際にアクセスを削除する必要があります。
- Microsoft のスタッフは、米国に残す前に、管理された環境のスマートカードを使用する必要があります。

**国内クラウド環境**:
- サードパーティのオペレーターまたはデータトラスティの担当者は、国内のクラウド環境へのアクセスの管理と適用を担当します。

Microsoft のクラウドサービス環境では、以下の表で説明するように、ユーザーの役割と関連するデータの種類に基づいてアクセスが制限されます。 米国外にある資格または資格のない人物は、米国のクラウド内の顧客データにアクセスすることを許可されていません。 国内のクラウド環境へのアクセスは、Microsoft の担当者が、サードパーティのオペレーターまたはデータのトラスティによる承認なしに顧客データや顧客データを含むシステムに対して技術的なアクセス権を持たないように制限されています。

|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------------------------|---------------------------------|

## <a name="microsoft-pre-employment-screening"></a>Microsoft 雇用前審査

現地の法律によって許可される場合、Microsoft のグローバルセキュリティ部門は雇用前審査を行います。 これは、次の条件を含む正式なバックグラウンド調査です。
- 申請者の経歴のチェック (たとえば、完全性と正確性)
- 学業および専門資格の確認
- プロフェッショナルな資格情報が失われた場合の調査
- 過去3人の雇用確認
- 州 conviction の警察レコードのチェック
- 自治体が発行した id からの id の確認
- 適切なクレジットチェック

特定の管理、セキュリティ、またはその他の役割 (顧客データへのアクセスを必要とする役割で米国をベースにしている従業員を含むが制限されていません) については、定期的な保護や追加のバックグラウンドチェックが必要になることがあります。
臨時スタッフの場合、サードパーティの契約では、サードパーティによって実施する必要がある Microsoft の審査要件を指定します。 バックグラウンドチェックの場合、サードパーティ企業は、バックグラウンドチェックが実行されたことを Microsoft 検証に提供する責任があります。 通常、バックグラウンドチェックの結果は、サードパーティの人事部からの電子メール経由で受信されます。 契約スタッフの国際的な従業員は、国の法律により、バックグラウンドチェックを禁止するため、バックグラウンドの審査プロセスから除外される場合があります。
