# <a name="eu-phone-number"></a>EU の電話番号

Office 365 のセキュリティでは、データ損失防止 (DLP)&amp;コンプライアンス センターには、DLP ポリシーに使用する準備ができている多くの機密性の高い情報の種類が含まれています。このトピックでは、EU の電話番号の機密性の高い情報の種類について説明します。 
  
## <a name="austria"></a>オーストリア

### <a name="format"></a>形式

標準の長さではありません。
  
### <a name="pattern"></a>パターン

- 桁 
    
- 携帯電話番号だけが数字の「6」で始まる 
    
### <a name="checksum"></a>チェックサム

該当なし
  
### <a name="definition"></a>定義

DLP ポリシーは、抽出した約 300 文字が次の条件に該当することを検出した場合に、75% の確証を持ってそれがこの種類の機密情報であると特定します。
  
- 正規表現`Regex_austria_eu_telephone_number_1`パターンに一致するコンテンツを検索します。 
    
- キーワードの`Keywords_austria_eu_telephone_number`があります。 
    
DLP ポリシーは、抽出した約 300 文字が次の条件に該当することを検出した場合に、75% の確証を持ってそれがこの種類の機密情報であると特定します。
  
- 正規表現`Regex_austria_eu_telephone_number_2`パターンに一致するコンテンツを検索します。 
    
- キーワードの`Keywords_austria_eu_telephone_number`があります。
