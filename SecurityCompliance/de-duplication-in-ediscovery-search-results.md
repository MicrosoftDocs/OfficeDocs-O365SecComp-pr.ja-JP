<span data-ttu-id="feeef-105">Office 365 の電子情報開示ツールを使用して電子情報開示検索の結果をエクスポートする場合、エクスポートされた結果を重複除外するオプションがあります。</span><span class="sxs-lookup"><span data-stu-id="feeef-105">When using Office 365 eDiscovery tools to export the results of an eDiscovery search, you have the option to de-duplicate the results that are exported.</span></span> <span data-ttu-id="feeef-106">シナリオ</span><span class="sxs-lookup"><span data-stu-id="feeef-106">What does this mean?</span></span> <span data-ttu-id="feeef-107">重複除去を有効にした場合 (既定では、重複除外は有効になっていません)、同じメッセージの複数のインスタンスが検索されたメールボックスで見つかった場合でも、電子メールメッセージのコピーは1つしかエクスポートされません。</span><span class="sxs-lookup"><span data-stu-id="feeef-107">When you enable de-duplication (by default, de-duplication isn't enabled), only one copy of an email message is exported even though multiple instances of the same message might have been found in the mailboxes that were searched.</span></span> <span data-ttu-id="feeef-108">重複除外は、検索結果のエクスポート後にレビューして分析する必要があるアイテムの数を減らすことで、時間を節約するのに便利です。</span><span class="sxs-lookup"><span data-stu-id="feeef-108">De-duplication helps you save time by reducing the number of items that you have to review and analyze after the search results are exported.</span></span> <span data-ttu-id="feeef-109">ただし、重複除外のしくみを理解しておくことは重要であり、エクスポート処理中に一意のアイテムが重複としてマークされる可能性があるアルゴリズムには制限があることに注意してください。</span><span class="sxs-lookup"><span data-stu-id="feeef-109">But it's important to understand how de-duplication works and be aware that there are limitations to the algorithm that might cause a unique item to be marked as a duplicate during the export process.</span></span>
  
## <a name="how-duplicate-messages-are-identified"></a><span data-ttu-id="feeef-110">重複するメッセージの識別方法</span><span class="sxs-lookup"><span data-stu-id="feeef-110">How duplicate messages are identified</span></span>

<span data-ttu-id="feeef-111">Office 365 電子情報開示ツールは、次の電子メールプロパティの組み合わせを使用して、メッセージが重複しているかどうかを判断します。</span><span class="sxs-lookup"><span data-stu-id="feeef-111">Office 365 eDiscovery tools use a combination of the following email properties to determine whether a message is a duplicate:</span></span>
  
- <span data-ttu-id="feeef-112">**internetmessageid** -このプロパティは、特定のメッセージの特定のバージョンを参照するグローバル一意識別子である、電子メールメッセージのインターネットメッセージ識別子を指定します。</span><span class="sxs-lookup"><span data-stu-id="feeef-112">**InternetMessageId** - This property specifies the Internet message identifier of an email message, which is a globally unique identifier that refers to a specific version of a specific message.</span></span> <span data-ttu-id="feeef-113">この ID は、送信者の電子メールクライアントプログラム、またはメッセージを送信するホスト電子メールシステムによって生成されます。</span><span class="sxs-lookup"><span data-stu-id="feeef-113">This ID is generated by the sender's email client program or host email system that sends the message.</span></span> <span data-ttu-id="feeef-114">ユーザーが複数の受信者にメッセージを送信した場合、そのメッセージの各インスタンスのインターネットメッセージ ID は同じになります。</span><span class="sxs-lookup"><span data-stu-id="feeef-114">If a person sends a message to more than one recipient, the Internet message ID will be the same for each instance of the message.</span></span> <span data-ttu-id="feeef-115">元のメッセージに対するその後の変更では、別のメッセージ識別子が受信されます。</span><span class="sxs-lookup"><span data-stu-id="feeef-115">Subsequent revisions to the original message will receive a different message identifier.</span></span>

- <span data-ttu-id="feeef-121">**bodytaginfo** -これは内部の Exchange ストアのプロパティです。</span><span class="sxs-lookup"><span data-stu-id="feeef-121">**BodyTagInfo** - This is an internal Exchange store property.</span></span> <span data-ttu-id="feeef-122">このプロパティの値は、メッセージ本文のさまざまな属性をチェックすることによって計算されます。</span><span class="sxs-lookup"><span data-stu-id="feeef-122">The value of this property is calculated by checking various attributes in the body of the message.</span></span> <span data-ttu-id="feeef-123">このプロパティは、メッセージ本文の違いを識別するために使用されます。</span><span class="sxs-lookup"><span data-stu-id="feeef-123">This property is used to identify differences in the body of messages.</span></span> 
    
<span data-ttu-id="feeef-124">電子情報開示のエクスポートプロセスでは、これらの3つのプロパティは、検索条件に一致するすべてのメッセージに対して比較されます。</span><span class="sxs-lookup"><span data-stu-id="feeef-124">During the eDiscovery export process, these three properties are compared for every message that matches the search criteria.</span></span> <span data-ttu-id="feeef-125">2つ以上のメッセージでこれらのプロパティが同一である場合、これらのメッセージは重複していると判断され、重複除外が有効になっている場合は、メッセージのコピーが1つだけエクスポートされることになります。</span><span class="sxs-lookup"><span data-stu-id="feeef-125">If these properties are identical for two (or more) messages, those messages are determined to be duplicates and the result is that only one copy of the message will be exported if de-duplication is enabled.</span></span> <span data-ttu-id="feeef-126">エクスポートされるメッセージは、"ソースアイテム" と呼ばれます。</span><span class="sxs-lookup"><span data-stu-id="feeef-126">The message that is exported is known as the "source item".</span></span> <span data-ttu-id="feeef-127">重複するメッセージに関する情報は、エクスポートされた検索結果に含まれる **.csv**レポートと**manifest.xml**レポートに含まれています。</span><span class="sxs-lookup"><span data-stu-id="feeef-127">Information about duplicate messages is included in the **Results.csv** and **Manifest.xml** reports that are included with the exported search results.</span></span> <span data-ttu-id="feeef-128">結果の **.csv**ファイルでは、重複するメッセージは、[**アイテムの複製**] 列に値があることで識別されます。</span><span class="sxs-lookup"><span data-stu-id="feeef-128">In the **Results.csv** file, a duplicate message is identified by having a value in the **Duplicate to Item** column.</span></span> <span data-ttu-id="feeef-129">この列の値は、エクスポートされたメッセージの [**アイテムの id** ] 列の値と一致します。</span><span class="sxs-lookup"><span data-stu-id="feeef-129">The value in this column matches the value in the **Item Identity** column for the message that was exported.</span></span> 
  
<span data-ttu-id="feeef-130">次の図は、検索結果と共にエクスポートされる**結果の .csv**レポートと**manifest.xml**レポートで重複メッセージがどのように表示されるかを示しています。</span><span class="sxs-lookup"><span data-stu-id="feeef-130">The following graphics show how duplicate messages are displayed in the **Results.csv** and **Manifest.xml** reports that are exported with the search results.</span></span> <span data-ttu-id="feeef-131">これらのレポートには、以前に説明した電子メールのプロパティは含まれません。これは、重複除外のアルゴリズムで使用されます。</span><span class="sxs-lookup"><span data-stu-id="feeef-131">These reports don't include the email properties previously described, which are used in the de-duplication algorithm.</span></span> <span data-ttu-id="feeef-132">代わりに、Exchange ストアによってアイテムに割り当てられる**アイテム id**プロパティがレポートに含まれます。</span><span class="sxs-lookup"><span data-stu-id="feeef-132">Instead, the reports include the **Item Identity** property that is assigned to items by the Exchange store.</span></span>
